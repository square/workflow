onPropsChanged

open fun onPropsChanged(old: PropsT, new: PropsT, state: StateT): StateT(source)

Called immediately before render if the parent workflow has provided updated PropsT that are not == to those provided previously. This allows the child to update its state based on the new information before rendering.

Note, though, that it is generally a mistake to copy information from PropsT to StateT! PropsT is always available via the renderProps parameter of render, and as props for a WorkflowAction. This method is mainly useful if you need to switch an enum or sealed StateT based on PropsT.

Default implementation does nothing.

Return

the new StateT. render is then called immediately with this value as renderState.
